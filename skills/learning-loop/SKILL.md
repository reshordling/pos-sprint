# Learning Loop

## Overview

`learning-loop` переводит сессию/разбор из заметок в управляемый план следующего шага.
Результат всегда пишется в тот же файл, где находится основной контекст.

## Quick Start

1. Определи целевой файл (например: `workshops/YYYY-MM-DD-topic.md`).
2. Найди или создай в нем раздел `## Learning Loop`.
3. Обнови 4 обязательных блока: `Саммари`, `Валидация тезисов`, `Задачи`, `Ссылки на артефакты`.
4. Проверь, что каждый новый тезис и задача подтверждены фактами/файлами.

## Когда использовать

- После воркшопа или сессии разбора.
- После изучения статьи/документа/треда, если нужно зафиксировать понимание.
- В любой момент, когда нужно обновить статус тезисов и список задач по конкретному файлу.

## Вход

- Целевой markdown-файл внутри POS.
- Фактические результаты текущей итерации: что сделано, какие файлы созданы/изменены, какие выводы подтверждены.

## Выход

- Обновленный раздел `## Learning Loop` в целевом файле.
- Четкий список следующих шагов в порядке выполнения.
- Актуальные ссылки на артефакты (только реально существующие).

## Поддерживаемые цели (примеры)

- `workshops/YYYY-MM-DD-topic.md`
- `projects/.../*.md`
- `threads/...md`
- любой markdown-файл внутри POS

## Формат раздела Learning Loop

1. `### Саммари`
2. `### Валидация тезисов`
   - `#### Понял`
   - `#### Не понял`
3. `### Задачи (по порядку выполнения)`
   - `#### Основной проект`
   - `#### Воркшоп/скиллы` (опционально)
4. `### Ссылки на артефакты`

## Алгоритм применения

1. Определи целевой файл (`target file`), где должен жить Learning Loop.
2. Открой файл и найди раздел `## Learning Loop`.
3. Если раздела нет — добавь его с полной структурой.
4. Если раздел уже есть — обнови пункты по фактам текущего состояния.
5. Переноси тезисы из `Не понял` в `Понял` только при наличии подтверждения (артефакт/результат).
6. Обновляй задачи в порядке выполнения, опциональные шаги помечай `(*)`.
7. Синхронизируй `Ссылки на артефакты` с фактически созданными/измененными файлами.

## Definition of Done

- Раздел `Learning Loop` существует и заполнен без пустых заглушек.
- В `Саммари` есть 3-7 практичных пунктов.
- В `Валидация тезисов` отражено текущее состояние (`Понял/Не понял`) без противоречий.
- В `Задачи` шаги упорядочены по исполнению; опциональные помечены `(*)`.
- В `Ссылки на артефакты` нет "битых" или несуществующих ссылок.

## Anti-Patterns

- Создавать отдельный `*-learning-loop.md` вместо обновления целевого файла.
- Переносить тезисы в `Понял` без подтверждающего результата.
- Оставлять задачи без порядка выполнения или без статуса.
- Добавлять в артефакты файлы, которые не были созданы/обновлены в итерации.

## Правила качества

1. Learning Loop обновляется в одном и том же целевом файле, где живет основной контекст.
2. Никаких дубликатов с отдельными `*-learning-loop.md`.
3. Каждый новый вывод должен быть проверяемым (факт, файл, результат).
4. Формулировки короткие, практичные, без лишней теории.
